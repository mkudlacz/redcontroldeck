

# Redcontroldeck — Brand Summary & Asset Usage Guide

## 1) What “Redcontroldeck” Is
**Redcontroldeck is an artist/maker’s imprint**, not a product brand.

- Origin: misheard lyric (“rent-controlled den” → “red control deck”)
- Longstanding alias used across: writing, music, events, online identity
- Concept: a **black box / control system** that produces and frames artifacts
- Tone: post‑modern, punk-adjacent, slightly panoptic (signal, control, observation)
- Role: a **mark of origin** applied to outputs (not a company explaining itself)

> Mental model: record label imprint / zine publisher / stamping device

---

## 2) Design Principles (non-negotiable)
- **Restraint over cleverness** (avoid gimmicks and over-design)
- **Separation of concerns** (wordmark, icon, and “deck” are distinct layers)
- **Repeatable minimal system** (few elements, used consistently)
- **Object + imprint duality** (device exists, but the imprint is primary)
- **Signal metaphor** (red dot / power light as recurring motif)

---

## 3) Core Assets (final set)

### A. Primary Wordmark
`REDCONTROLDECK` (often split RED / CONTROLDECK)

- Bold, condensed sans serif
- Clean, horizontal, confident
- No embedded graphics (no knobs-as-letters)
- Used for identification and authorship

**Recommended fonts**
- Druk Condensed (ideal)
- Trade Gothic Condensed Bold
- Helvetica Neue Condensed Black
- Fallbacks: Archivo Narrow, Oswald

---

### B. Icon System (minimal + fixed)
Keep to **two primary icons**:

1. **Signal Dot**
   - Solid red circle
   - Meaning: transmission / live / active system

2. **RCD Button**
   - Black circle with “RCD” knockout
   - Meaning: compact imprint / app identity

> Do not expand icon set unnecessarily. Power comes from repetition.

---

### C. Expressive “Deck” (poster / myth object)
- Vintage stereo/control deck aesthetic
- Black / cream / red palette
- Single red power light as focal point
- Slight grit, but controlled

**Usage**
- Poster
- Hero/landing visuals (sparingly)
- Special releases

> This is the **myth-object**. Do not use as a default logo.

---

### D. Color System
- `--rcd-red: #c62828`
- `--rcd-black: #111111`
- `--rcd-cream: #f4f1ea`
- `--rcd-signal: #ff3b30` (for active states)

---

## 4) Usage Hierarchy (how to combine assets)

| Layer | Asset | Purpose |
|------|------|--------|
| 1 | Deck (poster) | Myth / object / entry |
| 2 | Wordmark | Imprint / authorship |
| 3 | Icons | Signal / UI / system state |
| 4 | Projects list | Outputs |

**Do not collapse these into a single graphic.** Keep them distinct.

---

## 5) Index Page Pattern (implemented)

### Structure
1. **Poster (top)**
   - Represents the system/device

2. **(Optional) Wordmark**
   - Can be omitted or secondary

3. **Projects List**
   - Simple, horizontal list
   - Example:
     - *No Tape Left Behind Player*

4. **Tagline**
   - Plainspoken, descriptive (documentation tone)
   - Avoid marketing language

### Behavior
- Minimal interaction
- Single or few links
- No navigation, no explanation overload

> The page is a **threshold**, not a homepage.

---

## 6) Tone & Copy Guidelines
- Prefer **artifact language** over marketing
  - “player”, “collection”, “archive”
- Slightly technical / documentary voice
- Avoid hype, branding clichés, or UX copy patterns
- Feels like **notes about a system**, not promotion

---

## 7) Do / Don’t

### Do
- Use the wordmark cleanly and consistently
- Reuse the red signal dot across contexts
- Let the deck appear sparingly as a focal object
- Keep layouts tight and typeset (not “app-y”)
- Treat outputs as **projects**, not features

### Don’t
- Don’t integrate knobs or decorative elements into type
- Don’t create many icon variants
- Don’t overuse distress/grunge
- Don’t turn the site into a marketing homepage
- Don’t explain the concept too much

---

## 8) Application Notes (current project)

**Internet Archive Player**
- Header: wordmark (optional/minimal)
- Favicon: red dot or RCD button
- “Playing” state: signal dot / power light
- Index page: poster + project link
- Project label: “No Tape Left Behind Player”

---

## 9) Summary
Redcontroldeck functions as a **system that produces artifacts**.

Design expresses:
- **Object (deck)** → the machine
- **Imprint (wordmark)** → authorship
- **Signal (dot/icon)** → transmission
- **Projects** → outputs

The system is effective when:
> it feels discovered, consistent, and slightly opaque — not explained or over-designed.
